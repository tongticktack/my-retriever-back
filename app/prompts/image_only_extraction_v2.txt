# 이미지 단독 입력 분실물 추론 프롬프트 (v2)
# 목적: 사용자 텍스트 설명 없이 이미지(들)만 전달된 상황에서 분실물 핵심 필드 후보를 조심스럽게 추론.
# 출력: JSON 한 객체 ONLY (추가 문장/마크다운 금지).
# 언어: 반드시 한국어. 불확실하면 null (억지 추측 금지).
# 금지: 영어 혼용, 여분 텍스트, 코드 fences, 설명 문단.

지침:
1. 제공된 이미지를 관찰하여 분실물로 보이는 대상의 카테고리(대분류/소분류)와 대표 색상 한 가지를 식별.
2. 확실하지 않으면 해당 필드는 null 로 둔다.
3. region, lost_date 는 이미지로 확정 불가 → 항상 null.
4. 근거는 confidence_notes 하위 필드에 한국어 짧은 문장으로 기록. 낮은 확신이면 "추측"이라는 단어 포함.
5. need_more 배열에 사용자에게 꼭 추가로 질문해야 할 항목(예: 분실 지역, 날짜, 구체적 브랜드/모델 등)을 넣는다.
6. category 값은 다음 중 하나만: 전자기기, 의류, 가방, 지갑, 액세서리 (없으면 null)
7. subcategory 는 대응되는 세부분류(휴대폰, 태블릿, 노트북, 이어폰, 카메라, 자켓, 셔츠, 바지, 모자, 백팩, 토트백, 크로스백, 카드지갑, 장지갑, 동전지갑, 시계, 목걸이, 반지, 팔찌) 중 하나.
8. color 는 빨강, 파랑, 초록, 검정, 흰색, 회색, 노랑, 갈색 중 하나.
9. 패턴/다색일 때 가장 넓은 면적 또는 시각적으로 가장 지배적인 단일색만 선택. 애매하면 null.
10. 출력 외곽 공백 외 아무 것도 추가하지 않는다.

출력 JSON 스키마 예시(예시는 결과에 포함 금지):
{
  "category": "전자기기",
  "subcategory": "휴대폰",
  "color": "검정",
  "region": null,
  "lost_date": null,
  "confidence_notes": {
    "category": "전면 유리와 측면 버튼 형태로 전자기기",
    "subcategory": "카메라 배치가 휴대폰 전형",
    "color": "전체가 어두운 단색에 가까워 검정"
  },
  "need_more": ["어디에서 분실했나요?", "분실한 날짜(YYYY-MM-DD)는 언제인가요?"]
}

이제 위 스키마와 동일한 키 순서로 실제 추론 JSON 객체만 출력. 다른 내용 금지.
